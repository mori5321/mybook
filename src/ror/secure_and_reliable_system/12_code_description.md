## 一般的なセキュリティ脆弱性
- SQLインジェクション
- 認証の突破
- 機密データの漏洩
    - クレジットカード番号などの機密データを保存及び処理するためには文字列ではなく明確な型を利用する
        - 型で適切な暗号化を矯正する
- XML外部エンティティ(XXE) <- これなに?
- アクセス制御の突破
    - アプリケーション固有の問題であることが多いため注意を要する。
- セキュリティの設定ミス
- XSS
- 安全でないdeserialization <- これなに?
- 不十分なロギングとモニタリング
    - アドホックなロギングに頼るのではなく、リクエストや他のイベントを低レベルのライブラリで必要に応じて記録および監視すべき。
        - 前章のinterceptor参照
